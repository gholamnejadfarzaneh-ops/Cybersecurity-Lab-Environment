Zero Trust Ingress Architecture

Securing an Edge Node without Inbound Firewall Exceptions

This project demonstrates a production-grade Zero Trust Network Access (ZTNA) implementation. The goal was to host a web-facing portfolio on physical hardware (ThinkPad T480) while maintaining a strict "Never Trust, Always Verify" security posture.

Deployment Stack:
• OS: Ubuntu 22.04 LTS (Hardened)
• Web Server: Nginx (Reverse Proxy)
• Connectivity: Cloudflare Tunnel (cloudflared)
• DNS: Secured CNAME Flattening

Architectural Advantage

Traditional hosting requires Port Forwarding, which opens a direct hole in the network firewall and exposes the origin IP. In this architecture, I utilized an outbound-only Cloudflare Tunnel.

By establishing a persistent encrypted connection to the Cloudflare Edge, the server remains completely "dark" to the public internet. Attacks like DDoS and Direct-to-IP port scanning are mitigated because the origin IP is never broadcasted.

Governance & Compliance

From a GRC perspective, this setup satisfies the principle of Least Privilege. Access is restricted at the edge, and the attack surface is minimized to a single outbound daemon. This aligns with modern NIST 800-207 standards for Zero Trust environments.
