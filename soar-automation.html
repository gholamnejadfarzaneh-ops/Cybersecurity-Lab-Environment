Automated Threat Triage with Python

Domain: SOAR & Operations // Tools: Python, Tshark, Linux

The Objective

This project simulates a SOC workflow where raw network traffic is automatically ingested and analyzed to detect potential Denial of Service (DoS) or network scanning attempts. This mimics a SOAR playbook that reduces manual triage time.

1. Ingestion & Transformation

Using tshark, raw .pcap data is converted into a structured CSV format, isolating the source IP addresses for lightweight processing.

tshark -r traffic.pcap -T fields -e ip.src -E separator=, > traffic.csv

2. Detection Logic (Python)

I developed this script to perform frequency analysis on the ingested logs. It flags any IP exceeding a specific threshold as a security alert.

import csv from collections import Counter # Threshold for suspicious volume THRESHOLD = 30 def analyze(): with open('traffic.csv', 'r') as f: counts = Counter(row[0] for row in csv.reader(f) if row) for ip, count in counts.items(): if count > THRESHOLD: print(f"[!] ALERT: Suspicious activity from {ip}") analyze()

Business Impact

By automating the identification of anomalous traffic, the Mean Time to Detect (MTTD) is reduced from minutes to seconds. In a consulting context for Deloitte, this demonstrates an ability to translate raw technical data into actionable security intelligence.
