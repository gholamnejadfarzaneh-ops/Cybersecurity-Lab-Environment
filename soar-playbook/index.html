Architecting a Custom SOAR Playbook: Automated Threat Intelligence & Triage

Reducing MTTR through Python-driven API Enrichment and Logic-Based Filtering

Cybersecurity Automation / Python / GRC Strategy

1. The Executive Summary

In a modern SOC, 'Alert Fatigue' is the primary enemy of efficiency. This project demonstrates a modular approach to automating the initial triage of suspicious indicators, transforming raw URL data into actionable intelligence without manual intervention.

2. The Challenge: Signal vs. Noise

Analysts often spend hours manually cross-referencing URLs against threat databases. My goal was to build a system that handles this 'Level 1' task programmatically, allowing human experts to focus on complex investigations.

3. System Architecture & Workflow

The solution is built on a Linux (Ubuntu) environment using a Python-based connector. The workflow handles everything from Base64 data normalization to stateful logging.

4. Technical Implementation: The 'Smarter' Logic

"I deliberately avoided a binary 'Pass/Fail' logic. Instead, I implemented a Weighted Threshold. This allows the system to prioritize alerts where multiple vendors agree on the threat, while flagging 'edge cases' for human intervention, significantly reducing False Positives."

5. Security & Best Practices

Implementation included strict secret management via .gitignore to protect CTI (Cyber Threat Intelligence) API keys and the use of isolated virtual environments to ensure system stability.

6. Future Scalability

The next iteration of this project involves containerizing the script using Docker to allow for cloud-native deployment within a CI/CD pipeline.
